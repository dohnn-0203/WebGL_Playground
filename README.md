# 미니게임 천국 (Unity WebGL)

브라우저에서 바로 플레이하는 Unity WebGL 미니게임 모음입니다.
타이틀(게임 선택) 화면에서 게임을 고르면 시작되며, 현재 **머지 카페**, **수박게임**, **버블슈터** 세 가지를 플레이할 수 있습니다.

## 플레이

- **GitHub Pages**: https://dohnn-0203.github.io/WebGL_Playground/
  (저장소 Settings → Pages → Source를 `main` / `docs`로 설정하면 활성화됩니다)
- **로컬 실행**: 저장소 루트에서 `python -m http.server 8765 --directory docs` 후
  브라우저에서 `http://localhost:8765` 접속

## 게임 방법

### 1. 머지 카페 (머지 퍼즐)

커피·빵·디저트를 만들고, 같은 아이템 두 개를 합쳐 상위 메뉴를 제작해 주문을 납품하는 게임입니다.

1. 보드 위 **생성기**(커피머신 등)를 **탭**하면 인근 빈 칸에 Lv.1 아이템이 생성됩니다. 모든 생성기가 공유하는 **에너지**를 1 소모하며, 시간이 지나면 회복됩니다(5초당 +1).
2. 아이템을 **드래그**해 이동하고, 같은 종류·같은 레벨 위에 놓으면 **머지**되어 한 단계 높은 아이템이 됩니다. (각 계열 Lv.1~Lv.5) 생성기 타일도 드래그로 옮길 수 있습니다.
3. 좌측 **주문** 카드가 요구하는 아이템을 보드에 만들면 `납품` 버튼이 켜지고, 납품하면 **골드**를 받습니다.
4. 하단 바에서 골드로 **보드 확장**(잠긴 칸 해금)과 **에너지 강화**(최대 에너지↑)를 할 수 있습니다.
5. 좌측 상단 **"!"** 버튼으로 계열별 머지 조합표를 볼 수 있고, 진행 상황은 행동할 때마다 자동 저장됩니다(새로고침해도 유지). 상단 `초기화` 버튼으로 처음부터 시작합니다.

| 생성기 | 생산 아이템 |
|---|---|
| 커피머신 | 커피 계열 (원두 → 스페셜 커피) |
| 오븐 | 빵 계열 (반죽 → 디저트 플레이트) |
| 냉장고 | 디저트 계열 (크림 → 시그니처 케이크) |

> 세 생성기는 **하나의 에너지 게이지를 공유**합니다. 기본 최대 20, 5초당 +1 회복, 강화로 최대치를 늘립니다.

### 2. 수박게임 (스이카 게임)

과일을 떨어뜨려 같은 과일끼리 합치고, 마지막 **수박**까지 키우는 물리 퍼즐입니다.

1. **마우스 왼쪽 클릭한 위치에서 과일이 아래로 떨어집니다.** 마우스를 좌우로 움직여 떨어질 자리를 조준하세요.
2. 과일은 **중력으로 굴러가며 쌓입니다.** 같은 과일 두 개가 닿으면 사라지고 **한 단계 큰 과일**이 됩니다.
3. 과일 진행 순서(작은 → 큰, 총 11종):
   **체리 → 딸기 → 포도 → 귤 → 오렌지 → 사과 → 배 → 복숭아 → 파인애플 → 멜론 → 수박**
4. 떨어뜨릴 수 있는 건 작은 과일 5종(체리~오렌지)이고, **다음에 나올 과일**은 우측 상단에 미리 표시됩니다.
5. 합칠수록, 큰 과일을 만들수록 **점수**가 올라갑니다. 연쇄(콤보)를 노려보세요.
6. 과일이 통 위쪽 **빨간 경계선을 넘으면 게임 오버**입니다. 점수와 함께 `다시하기` / `메뉴`가 표시됩니다.
7. 좌측 상단 `메뉴` 버튼으로 게임 선택 화면으로 돌아갑니다.

> 팁: 같은 과일을 가까이 모아 두고, 큰 과일은 아래·가장자리로 유도해 높이를 낮게 유지하세요.

### 3. 버블슈터 (Bubble Shooter)

아래에서 색 구슬을 발사해 같은 색 **3개 이상**을 붙여 터뜨리는 고전 퍼즐입니다.

1. **마우스를 움직여 조준**하면 발사 경로(벽 반사 포함)가 점선으로 미리 표시됩니다.
2. **왼쪽 클릭으로 발사**합니다. 구슬은 벽에 튕기며 날아가 천장이나 다른 구슬에 닿으면 **육각 격자**에 달라붙습니다.
3. 같은 색 구슬이 **3개 이상 연결**되면 **터지고 점수**를 얻습니다. 터진 구슬 아래로 천장과 끊긴 구슬 뭉치는 **떨어지며 보너스 점수**(구슬당 더 높음)를 줍니다.
4. 다음에 발사할 구슬 색은 좌측 상단에 미리 표시됩니다.
5. 모든 구슬을 없애면 **클리어**, 구슬이 아래 경계선까지 쌓이면 **게임 오버**입니다.
6. 좌측 상단 `메뉴` 버튼으로 게임 선택 화면으로 돌아갑니다.

> 팁: 벽 반사를 이용하면 직선으로 닿지 않는 구석도 노릴 수 있습니다. 매달린 뭉치의 받침을 끊으면 크게 떨어뜨려 점수를 극대화할 수 있어요.

## 개발 정보

- Unity `2022.3.62f3`, 2D 템플릿, 순수 Unity 기능만 사용 (외부 유료/미확인 라이선스 에셋 없음)
- 씬은 `Assets/Scenes/MergeCafeGame.unity` 하나이며, UI와 게임을 **코드로 런타임 구성**
- 게임별로 어셈블리(asmdef)를 분리: `Common`(공용 UI·아트) ← `MergeCafe`/`Suika`/`BubbleShooter` ← `App`(허브)
- 모든 아이콘/아트는 **직접 제작한 절차적 2D 그래픽** (라이선스 클린)
- 텍스트는 **TextMeshPro(SDF)**, 한글은 Noto Sans KR(SIL OFL 1.1) 기반 다이내믹 폰트를 런타임 생성
  (고지: `Assets/Common/Docs/THIRD_PARTY_NOTICES.md`)
- 수박게임은 Unity **2D 물리**(Rigidbody2D·Collider2D), 버블슈터는 **육각 격자 + 수동 궤적 물리** 사용
- 게임 로직은 순수 C# — 자동화 테스트 **EditMode 80건 + PlayMode 6건**
- 설계 문서: [`webGL_game.md`](webGL_game.md)

### 빌드

에디터 메뉴 `MergeCafe → Build WebGL (docs)` 또는 배치 모드:

```text
Unity.exe -batchmode -nographics -quit -projectPath <프로젝트 경로> ^
  -buildTarget WebGL -executeMethod MergeCafe.EditorTools.WebGLBuilder.BuildForGitHubPages
```

출력은 저장소 루트 `docs/`. **압축은 Disabled(무압축)** 로 빌드합니다 — GitHub Pages는 사전 압축된 `.unityweb` 전송을 제대로 처리하지 못해 런타임 크래시가 발생하므로, 무압축으로 배포하고 Pages가 자체 transparent gzip으로 전송하게 합니다.
